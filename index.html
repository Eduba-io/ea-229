01 · The Eduba frame

60 / 30 / 10, applied to an AI-infrastructure field motion.

Most organizations automate the wrong things. About 60% of work is traditional code and database. About 30% is rule-based logic. About 10% is genuine AI. Applied to how a NetApp seller or SE runs the first 20 minutes of a greenfield AI meeting, the map is specific.

60%

Database and code. The reference architecture.

Instance-level positioning against Pure, VAST, and hyperscaler first-party alternatives. ONTAP facts. FSx for NetApp ONTAP, Azure NetApp Files, Google Cloud NetApp Volumes. This is recall and muscle memory. Curriculum, not consulting.

30%

Rule-based logic. The qualification frame.

When is AFX the right call. When is ONTAP already enough. When does BlueXP classification land ahead of the infrastructure conversation. A short decision tree the SE can run in under five minutes. Reusable playbook.

10%

Genuine AI. The customer's real problem.

The customer is not buying storage. They are trying to get an LLM, a RAG pipeline, or an agent workflow into production without burning a data-governance audit. This is the methodology conversation. This is where the seller earns the next meeting.

03 · The underlying research

Interpretable Context Methodology.

Submitted to ACM Transactions on Interactive Intelligent Systems. The argument: agent context can be organized as a layered filesystem (L0 identity through L4 working artifacts) with measurable gains in interpretability and reproducibility. If BlueXP classification is the map of where data lives, ICM is the methodology that turns the map into a working operating model for an AI-capable enterprise.

Repo: github.com/RinDig/Interpretable-Context-Methodology-ICM-. MIT license.

Companion work: Ethics Engine (arXiv:2510.11742). A psychometric assessment tool for evaluating ideological and moral patterns in LLMs. Relevant when an AI workflow stands inside a regulated data environment. github.com/RinDig/AuditEngine.
